Quick SEO: A Comprehensive Breakdown of Features and Functional Architecture

Quick SEO is a modular, WordPress-native plugin designed to provide granular control over SEO fields for WordPress and WooCommerce websites. It prioritizes transparency, local control, and developer-friendly operations with zero dependency on cloud services or third-party APIs.

1. SEO Metadata Injection

Quick SEO supports injection of <title>, meta description, and meta keywords into the head of each post, page, product, or taxonomy archive. These values can be edited manually or via spreadsheet, and are saved via WordPress‚Äôs post_meta or term_meta APIs.

2. Open Graph and Twitter Card Support

Includes support for OG tags like og:title, og:description, og:image, and Twitter card tags such as twitter:title and twitter:image. Defaults cascade from primary SEO fields unless explicitly overridden.

3. Canonical URL Controls

Canonical URLs can be set per page or via global logic. If no override is present, the plugin falls back to WordPress's permalink structure. Canonical values are injected via <link rel="canonical"> using wp_head output buffering.

4. Robots Meta Tag and Robots.txt Editor

Per-object robots directives can be toggled using dropdown fields. Additionally, a GUI editor allows direct editing of the robots.txt file with save fallback for servers without write permissions.

5. JSON-LD Schema Builder

The schema system includes a builder interface for injecting JSON-LD schema types like WebPage, Article, Product, Organization, and more. Templates can be saved and reused across posts and taxonomies. Output is injected as <script type="application/ld+json">.

6. Longtail Keyword Injection

Invisible longtail blocks are injected using display: none containers in the body. These keywords are randomized per load (if multiple defined), stored in a custom field, and used to expand search coverage without cluttering content.

7. Spreadsheet-Style Bulk Editing

Each major object type (posts, pages, products, taxonomies) includes a spreadsheet interface for SEO data. Editable columns include:

  • Title
  • Description
  • Keywords
  • OG and Twitter Tags
  • Robots Directive
  • Canonical URL
  • Schema Template

Changes are saved via AJAX in batch mode to reduce load times.

8. Sitemap Generator

The plugin automatically generates a compliant sitemap.xml file including pages, posts, products, and taxonomy archives. It supports modified timestamps, priority rules, and change frequency configuration.

9. Conflict and Duplication Prevention

Quick SEO removes native WordPress or conflicting plugin outputs (e.g. Yoast) to prevent duplicate tags. Canonical conflicts are logged, and output is wrapped in visual comments for inspection.

10. SEO Meta Box

An SEO metabox is present on all relevant edit screens, including live character counts, OG/Twitter image upload, schema selection, and field validation indicators. Data is autosaved with the post.

11. Schema Fallback Logic

If no schema is defined, Quick SEO automatically falls back to a default per post type, then to a global fallback. A basic WebPage schema is always included if all else fails.

12. Structured Comments

When enabled, a <!-- QUICK SEO START --> block will be output in the document head showing all injected field values, schema type, and source attribution. This aids in debugging.

13. Local-Only Philosophy

Quick SEO does not rely on cloud services. All settings, fields, schema, and files remain local. There are no external API calls, tracking scripts, or upsells. Export and import options use JSON files for full transparency.

14. Developer Focused Design

The plugin includes proper filters, actions, and hooks throughout. JS and CSS are only loaded where necessary. All logic follows WP Coding Standards. AJAX endpoints are secured and modularized under includes/.

Here you can define global defaults for schema, robots, canonical rules, and whether to display structured HTML comments in page source.

II. SEO Metadata

Title, Description, Keywords

Set manually per page or fallback to permalink. Override logic set under Settings ‚Üí Canonical to enforce trailing slash policy.

V. Robots Meta and robots.txt

Each post/page/product can use:

VI. Schema Builder

  • Go to Quick SEO ‚Üí Schema
  • Create a schema template (e.g. Product, Organization)
  • Assign via metabox dropdown or spreadsheet

Injection is handled using valid JSON-LD in a script tag. If none is selected, fallback schema (WebPage) is applied.

VII. Longtail Keyword Blocks

Insert hidden semantic keyword phrases:

  • Each block is wrapped in <div style="display:none">

Quick SEO prevents duplication of:
